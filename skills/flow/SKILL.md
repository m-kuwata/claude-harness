# /flow — ワークフロー宣言

## 実行

```bash
bash ${CLAUDE_PLUGIN_ROOT}/scripts/flow-start.sh <workflow> [チケット番号]
```

引数なしで実行すると定義済みワークフローの一覧が出る。

## 手順

1. ユーザーの依頼がどのワークフローに当たるか判断する
   - 実装・修正 → `implement`（等のデフォルトフロー）。チケット番号が必要
   - コード変更を伴わない調査・分析 → read-only の調査系フロー
   - PR レビュー → PR レビュー系フロー（PR 番号を渡す）
   - 判断に迷う場合はユーザーに確認する
2. チケットが必要なフローでチケットが存在しない場合は、先にチケットを作成する
   （プロジェクトの issue 運用ルールに従う）
3. `flow-start.sh` を実行し、出力された**ゲート計画を読む**
4. entry ゲート（例: /tdd）があれば着手前に完了する
5. 以降は Stop フックの指示に従う。各ゲートの完了記録は
   block メッセージに埋め込まれた**トークン付きコマンド**でのみ行える

## してはいけないこと

- トークンなしで `mark-gate-passed.sh` を実行する（無効）
- マーカーファイルを直接作成・編集する（無効 + 禁止）
- ゲートスキルを実行せずに完了記録だけ行う
